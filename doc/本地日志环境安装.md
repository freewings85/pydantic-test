# 本地日志环境安装指南

## 架构

```
Python 应用 (logfire SDK)
    │  OTLP HTTP (:4318)
    v
OTel Collector         ← 接收所有 OTel 数据，路由分发
    ├──→ Tempo          ← 存储 traces（调用链）
    └──→ Loki           ← 存储 logs（日志）
         │
Grafana  ←─────────────── Web UI，查询 traces + logs
```

## 前置条件

- Docker + Docker Compose

## 安装步骤

### 1. 配置文件

项目中已包含所有配置文件，无需额外创建：

```
otel-config/
├── otel-collector.yml        # OTel Collector 配置（接收 → 路由）
├── tempo.yml                 # Tempo 配置（traces 存储）
├── loki.yml                  # Loki 配置（logs 存储）
└── grafana-datasources.yml   # Grafana 数据源自动配置
```

### 2. 启动服务

```bash
cd /mnt/e/Documents/github/pydantic-test
docker compose up -d
```

启动 4 个容器：

| 容器 | 镜像 | 端口 | 作用 |
|------|------|------|------|
| otel-collector | `otel/opentelemetry-collector-contrib:latest` | 4317 (gRPC), 4318 (HTTP) | 接收 OTel 数据 |
| tempo | `grafana/tempo:2.7.2` | 3200 | Traces 存储 |
| loki | `grafana/loki:latest` | 3100 | Logs 存储 |
| grafana | `grafana/grafana:latest` | 3000 | Web UI |

### 3. 验证启动

```bash
# 查看所有容器状态
docker compose ps

# 应该看到 4 个容器都是 running
```

### 4. 访问 Grafana

浏览器打开 http://localhost:3000

- 已配置免登录（匿名 Admin 权限）
- 数据源已自动配置（Tempo + Loki）

## 使用方式

### 产生 Trace 数据

```bash
uv run python main.py
```

### 在 Grafana 中查看

1. 打开 http://localhost:3000
2. 左侧菜单点 **Explore**
3. 数据源选 **Tempo**
4. Query type 选 **Search**，点 **Run query**
5. 点击某条 trace 查看瀑布图

### Python 端配置

```python
import logfire
from opentelemetry.sdk.trace.export import SimpleSpanProcessor
from opentelemetry.exporter.otlp.proto.http.trace_exporter import OTLPSpanExporter

logfire.configure(
    send_to_logfire=False,  # 不发到 Logfire 平台
    additional_span_processors=[
        SimpleSpanProcessor(
            OTLPSpanExporter(endpoint="http://localhost:4318/v1/traces")
        )
    ],
)
```

## 常用操作

```bash
# 启动
docker compose up -d

# 停止
docker compose down

# 停止并清除数据
docker compose down -v

# 查看日志（排查问题）
docker compose logs otel-collector
docker compose logs tempo
docker compose logs loki

# 重建某个容器（配置文件改了之后）
docker compose rm -f otel-collector && docker compose up -d otel-collector
```

## 踩坑记录

| 问题 | 解决方案 |
|------|----------|
| Tempo `empty ring` 错误 | 必须用 `grafana/tempo:2.7.2`，不要用 latest（v2.10+ 单节点不兼容） |
| Loki `permission denied` | docker-compose 中 loki 设置 `user: "0"` |
| OTel Collector `loki` exporter 找不到 | 用 `otlphttp/loki` + endpoint `http://loki:3100/otlp` |
| 修改 otel-collector 配置后不生效 | `docker compose rm -f otel-collector` 再 `docker compose up -d` |
