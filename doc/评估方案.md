# Pydantic Evals 评估方案

## 概述

pydantic-evals 是 pydantic-ai 内置的评估框架，用于系统性地评估 LLM 应用的输出质量和执行行为。核心思路是**像写单元测试一样写评估**，但评估的是 LLM 应用的"质量"而非简单的"对错"。

**不依赖 pytest**，直接 `python eval.py` 就能跑。pytest 只是可选的 CI 集成方式。

## 核心数据模型

```
Dataset (测试集)
  └── Case (测试用例) × N
        ├── inputs           → 输入
        ├── expected_output  → 预期输出（可选）
        └── metadata         → 元数据标签

Task (被测任务)
  └── 你的 agent/函数，接收 inputs 返回 output

Evaluator (评估器) × N
  └── 对每个 Case 的 output 打分

Dataset.evaluate_sync(task)
  └── 遍历所有 Case → 执行 Task → 运行 Evaluators → 生成 Report
```

## 三种评估器类型

### 1. 代码评估器 — 确定性检查

用代码逻辑判断输出是否满足条件，成本低、结果确定。

```python
from dataclasses import dataclass
from pydantic_evals.evaluators import Evaluator, EvaluatorContext

@dataclass
class ContainsTemperature(Evaluator[str, str]):
    """检查回答中是否包含温度信息"""
    async def evaluate(self, ctx: EvaluatorContext[str, str]) -> dict:
        output = ctx.output.lower()
        has_temp = "°c" in output or "°f" in output
        return {"contains_temperature": has_temp}

@dataclass
class MentionsCity(Evaluator[str, str]):
    """检查回答中是否提到了被问到的城市"""
    async def evaluate(self, ctx: EvaluatorContext[str, str]) -> dict:
        city = ctx.expected_output.lower() if ctx.expected_output else ""
        return {"mentions_city": city in ctx.output.lower()}
```

**适用场景**：格式校验、关键词检查、类型匹配、精确匹配。

### 2. LLM Judge — 主观质量评估

用另一个 LLM 对输出进行打分，适合需要"理解"和"判断"的场景。

```python
from pydantic_evals.evaluators import LLMJudge

LLMJudge(
    rubric=(
        "The response should: "
        "1) Mention the specific city asked about. "
        "2) Include temperature information. "
        "3) Be concise and natural. "
        "4) Not contain hallucinated or made-up weather data."
    ),
    include_input=True,
    model=model,  # 可以指定用哪个模型做评判
)
```

**LLMJudge 参数说明**：

| 参数 | 说明 |
|------|------|
| `rubric` | 评分标准，越具体越好 |
| `include_input` | 是否把原始输入也给 judge 看 |
| `include_expected_output` | 是否把预期输出也给 judge 看 |
| `model` | 用哪个模型做评判，默认 OpenAI，可指定为 Azure 等 |
| `assertion` | 返回 pass/fail 断言（默认开启） |
| `score` | 返回 0.0-1.0 连续分数（默认关闭） |

**适用场景**：准确性、有用性、幻觉检测、语气风格、RAG 接地性检查。

**注意**：LLMJudge 默认用 OpenAI 模型打分。如果用 Azure OpenAI，需要显式传入 `model` 参数，否则会报 `OPENAI_API_KEY` 缺失错误。

### 3. Span-Based 评估器 — 检查执行过程

基于 OpenTelemetry span 分析 agent 的**执行行为**，而非仅看最终输出。需要配合 logfire 使用。

```python
from pydantic_evals.evaluators import HasMatchingSpan

# 验证 agent 调用了 get_weather tool
HasMatchingSpan(
    query={"has_attributes": {"gen_ai.tool.name": "get_weather"}},
    evaluation_name="called_weather_tool",
)

# 验证有 HTTP 请求发出
HasMatchingSpan(
    query={"name_equals": "GET"},
    evaluation_name="made_http_request",
)

# 安全性检查：确认没有 DELETE 操作
HasMatchingSpan(
    query={"not_": {"name_contains": "DELETE"}},
    evaluation_name="no_delete_operations",
)
```

**`HasMatchingSpan` 参数**：

| 参数 | 说明 |
|------|------|
| `query` | SpanQuery 字典，框架定义的匹配规则 |
| `evaluation_name` | 自定义名称，显示在报告里，随便取 |

**适用场景**：验证 tool 调用、HTTP 请求行为、性能约束、安全性审计。

## SpanQuery 完整字段参考

SpanQuery 是框架定义的，所有字段都是可选的，默认 AND 组合。

### 名称匹配

| 字段 | 类型 | 说明 |
|------|------|------|
| `name_equals` | str | 精确匹配 span 名 |
| `name_contains` | str | 包含子串 |
| `name_matches_regex` | str | 正则匹配 |

### 属性匹配

| 字段 | 类型 | 说明 |
|------|------|------|
| `has_attributes` | dict | 键值对匹配 |
| `has_attribute_keys` | list[str] | 只检查 key 存在 |

### 耗时约束

| 字段 | 类型 | 说明 |
|------|------|------|
| `min_duration` | float / timedelta | 最小耗时 |
| `max_duration` | float / timedelta | 最大耗时 |

### 层级关系

| 字段 | 说明 |
|------|------|
| `some_child_has` | 至少一个子 span 满足 |
| `all_children_have` | 所有子 span 都满足 |
| `no_child_has` | 没有子 span 满足 |
| `some_descendant_has` | 至少一个后代 span 满足 |
| `some_ancestor_has` | 至少一个祖先 span 满足 |
| `min_child_count` / `max_child_count` | 子 span 数量 |
| `min_depth` / `max_depth` | span 嵌套深度 |

### 逻辑组合

| 字段 | 说明 |
|------|------|
| `and_` | 所有条件都满足 |
| `or_` | 任一条件满足 |
| `not_` | 取反 |

## Pydantic AI 的 Span 命名规则

通过查看实际的 trace 数据，发现 pydantic-ai 的 span 命名规则是：

**span name 是通用名，具体信息在 attributes 里。**

| Span Name | 关键 Attributes | 说明 |
|-----------|-----------------|------|
| `agent run` | — | agent 运行的根 span |
| `chat gpt-5-chat` | — | LLM 调用 |
| `running tools` | — | tool 执行的父 span（复数） |
| `running tool` | `gen_ai.tool.name`、`tool_arguments`、`tool_response` | 单个 tool 执行 |
| `GET` / `POST` | `http.url`、`http.method`、`http.status_code` | HTTP 请求 |

**重要**：写 span 评估器时，优先用 `has_attributes` 匹配具体工具/请求，而非 `name_contains`。

```python
# 错误：span 名是 "running tool"，不含 "get_weather"
HasMatchingSpan(query={"name_contains": "get_weather"}, ...)

# 正确：通过 attribute 匹配
HasMatchingSpan(query={"has_attributes": {"gen_ai.tool.name": "get_weather"}}, ...)
```

## 评估报告输出

### 终端打印

```python
report = dataset.evaluate_sync(weather_task)
report.print(include_input=True, include_output=True)
```

### 输出到文本文件

```python
txt_content = report.render(include_input=True, include_output=True)
Path("report.txt").write_text(txt_content)
```

### 输出到 JSON 文件

框架没有内置 `to_json()`，需要手动序列化：

```python
import json

json_data = {
    "name": report.name,
    "cases": [
        {
            "name": c.name,
            "inputs": c.inputs,
            "output": c.output,
            "scores": {k: v for k, v in (c.scores or {}).items()},
            "assertions": {
                k: v.value if hasattr(v, "value") else str(v)
                for k, v in (c.assertions or {}).items()
            },
            "metrics": {k: v for k, v in (c.metrics or {}).items()},
            "duration": float(c.task_duration) if c.task_duration else None,
        }
        for c in report.cases
    ],
}
Path("report.json").write_text(json.dumps(json_data, indent=2, ensure_ascii=False, default=str))
```

### 报告内容说明

| 字段 | 说明 |
|------|------|
| `Assertions` | ✔/✗ 列表，每个评估器一个结果 |
| `Metrics` | requests（LLM 请求次数）、input/output tokens |
| `Duration` | 单个 case 的执行耗时 |
| `Averages` | 所有 case 的平均值和通过率 |

## 完整示例

```python
from dataclasses import dataclass
from pydantic_evals import Case, Dataset
from pydantic_evals.evaluators import (
    Evaluator, EvaluatorContext, HasMatchingSpan, LLMJudge,
)
from main import agent, model


@dataclass
class ContainsTemperature(Evaluator[str, str]):
    async def evaluate(self, ctx: EvaluatorContext[str, str]) -> dict:
        output = ctx.output.lower()
        has_temp = "°c" in output or "°f" in output
        return {"contains_temperature": has_temp}


dataset = Dataset(
    cases=[
        Case(
            name="beijing_weather",
            inputs="What's the weather like in Beijing?",
            expected_output="Beijing",
        ),
    ],
    evaluators=[
        # 代码评估器
        ContainsTemperature(),

        # LLM Judge
        LLMJudge(
            rubric="Response should mention the city and include temperature.",
            include_input=True,
            model=model,
        ),

        # Span 评估器
        HasMatchingSpan(
            query={"has_attributes": {"gen_ai.tool.name": "get_weather"}},
            evaluation_name="called_weather_tool",
        ),
    ],
)


async def weather_task(inputs: str) -> str:
    result = await agent.run(inputs)
    return result.output


if __name__ == "__main__":
    report = dataset.evaluate_sync(weather_task)
    report.print(include_input=True, include_output=True)
```

运行：

```bash
uv run python eval.py
```

## 踩坑记录

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| LLMJudge 报 `OPENAI_API_KEY` 缺失 | 默认用 OpenAI 模型打分 | 显式传入 `model=model`（你的 Azure 模型） |
| `called_weather_tool` span 评估失败 | span 名是 `running tool`，不含工具名 | 改用 `has_attributes: {"gen_ai.tool.name": "get_weather"}` |
| `task_duration.total_seconds()` 报错 | `task_duration` 已经是 float，不是 timedelta | 直接 `float(c.task_duration)` |
| 中文城市名查询失败 | geocoding API 不支持中文 | 改为英文输入或在 tool 中做翻译 |
