# Pydantic AI 日志与可观测性方案

## 概述

Pydantic AI 通过 Logfire SDK 生成 OpenTelemetry (OTel) 标准格式的 trace 数据，后端可自由替换。本方案记录了三种已验证的方案，从开发调试到企业级生产环境逐步递进。

## 架构原理

```
Python 应用 (pydantic-ai + logfire SDK)
    |
    | OTLP HTTP/gRPC (标准协议)
    v
OTel Collector (数据路由)
    |
    +---> Tempo (traces 存储)
    +---> Loki (logs 存储)
    |
    v
Grafana (Web UI 可视化)
```

核心要点：
- `logfire.instrument_pydantic_ai()` 自动为 agent run、LLM 调用、tool 执行生成 span
- `logfire.instrument_httpx()` 自动为 HTTP 请求生成 span
- `send_to_logfire=False` 时不发送数据到 Pydantic 的 Logfire 平台，仅发到自定义后端
- 切换后端只需修改 OTLPExporter 的 endpoint 地址，应用代码无需改动

## 方案一：otel-tui（终端调试）

适用场景：本地开发快速调试，无需浏览器。

### docker-compose.yml

```yaml
services:
  otel-tui:
    image: ymtdzzz/otel-tui:latest
    stdin_open: true
    tty: true
    ports:
      - "4317:4317"
      - "4318:4318"
```

### Python 配置

```python
from opentelemetry.sdk.trace.export import SimpleSpanProcessor
from opentelemetry.exporter.otlp.proto.http.trace_exporter import OTLPSpanExporter

logfire.configure(
    send_to_logfire=False,
    additional_span_processors=[
        SimpleSpanProcessor(
            OTLPSpanExporter(endpoint="http://localhost:4318/v1/traces")
        )
    ],
)
```

### 查看方式

```bash
docker attach <container-name>
```

## 方案二：Logfire 平台（SaaS 托管）

适用场景：快速上手，无需自建基础设施。

### Python 配置

```python
logfire.configure()  # 默认发送到 Logfire 平台
logfire.instrument_pydantic_ai()
```

### 查看方式

访问 https://logfire-us.pydantic.dev/ 登录查看。需提前通过 `logfire auth` 完成认证。

## 方案三：Grafana + Tempo + Loki（企业级自托管）

适用场景：企业生产环境，需要 traces + logs 全栈可观测、告警、Dashboard、权限管理。

### 组件说明

| 组件 | 作用 | 端口 |
|------|------|------|
| OTel Collector | 接收 OTel 数据，路由到 Tempo 和 Loki | 4317 (gRPC), 4318 (HTTP) |
| Tempo | 存储和查询 traces | 3200 (内部) |
| Loki | 存储和查询 logs | 3100 (内部) |
| Grafana | Web UI，可视化 traces/logs | 3000 |

### docker-compose.yml

```yaml
services:
  otel-collector:
    image: otel/opentelemetry-collector-contrib:latest
    volumes:
      - ./otel-config/otel-collector.yml:/etc/otelcol-contrib/config.yaml
    ports:
      - "4317:4317"
      - "4318:4318"
    depends_on:
      - tempo
      - loki

  tempo:
    image: grafana/tempo:2.7.2
    volumes:
      - ./otel-config/tempo.yml:/etc/tempo.yaml
      - tempo-data:/var/tempo
    ports:
      - "3200:3200"
    command: ["-config.file=/etc/tempo.yaml"]

  loki:
    image: grafana/loki:latest
    user: "0"
    volumes:
      - ./otel-config/loki.yml:/etc/loki/local-config.yaml
      - loki-data:/var/loki
    command: ["-config.file=/etc/loki/local-config.yaml"]

  grafana:
    image: grafana/grafana:latest
    ports:
      - "3000:3000"
    volumes:
      - ./otel-config/grafana-datasources.yml:/etc/grafana/provisioning/datasources/datasources.yml
      - grafana-data:/var/lib/grafana
    environment:
      - GF_AUTH_ANONYMOUS_ENABLED=true
      - GF_AUTH_ANONYMOUS_ORG_ROLE=Admin
      - GF_AUTH_DISABLE_LOGIN_FORM=true
    depends_on:
      - tempo
      - loki

volumes:
  tempo-data:
  loki-data:
  grafana-data:
```

### 配置文件

#### otel-config/otel-collector.yml

```yaml
receivers:
  otlp:
    protocols:
      grpc:
        endpoint: 0.0.0.0:4317
      http:
        endpoint: 0.0.0.0:4318

exporters:
  otlphttp/tempo:
    endpoint: http://tempo:4318
  otlphttp/loki:
    endpoint: http://loki:3100/otlp

processors:
  batch:

service:
  pipelines:
    traces:
      receivers: [otlp]
      processors: [batch]
      exporters: [otlphttp/tempo]
    logs:
      receivers: [otlp]
      processors: [batch]
      exporters: [otlphttp/loki]
```

#### otel-config/tempo.yml

```yaml
stream_over_http_enabled: true
server:
  http_listen_port: 3200

distributor:
  receivers:
    otlp:
      protocols:
        http:
          endpoint: 0.0.0.0:4318
        grpc:
          endpoint: 0.0.0.0:4317

ingester:
  lifecycler:
    ring:
      replication_factor: 1

storage:
  trace:
    backend: local
    wal:
      path: /var/tempo/wal
    local:
      path: /var/tempo/blocks
```

#### otel-config/loki.yml

```yaml
auth_enabled: false

server:
  http_listen_port: 3100

common:
  path_prefix: /var/loki
  storage:
    filesystem:
      chunks_directory: /var/loki/chunks
      rules_directory: /var/loki/rules
  replication_factor: 1
  ring:
    kvstore:
      store: inmemory

schema_config:
  configs:
    - from: "2024-01-01"
      store: tsdb
      object_store: filesystem
      schema: v13
      index:
        prefix: index_
        period: 24h

limits_config:
  allow_structured_metadata: true
```

#### otel-config/grafana-datasources.yml

```yaml
apiVersion: 1

datasources:
  - name: Tempo
    type: tempo
    access: proxy
    url: http://tempo:3200
    isDefault: true
    jsonData:
      tracesToLogsV2:
        datasourceUid: loki
        filterByTraceID: true
      nodeGraph:
        enabled: true

  - name: Loki
    uid: loki
    type: loki
    access: proxy
    url: http://loki:3100
    jsonData:
      derivedFields:
        - datasourceUid: tempo
          matcherRegex: "trace_id=(\\w+)"
          name: TraceID
          url: "${__value.raw}"
```

### Python 配置

与方案一相同，OTLPExporter endpoint 指向 `http://localhost:4318/v1/traces`。

### 使用方式

```bash
# 启动服务
docker compose up -d

# 运行应用产生 trace
uv run main.py

# 浏览器访问 Grafana
# http://localhost:3000
# Explore -> Tempo -> Search -> Run query
```

### Grafana 中查看 Trace

1. 打开 http://localhost:3000（已配置免登录）
2. 左侧菜单点 Explore
3. 数据源选 Tempo
4. Query type 选 Search，点 Run query
5. 点击 trace 查看瀑布图，包含完整调用链：
   - agent run (总 span)
   - chat gpt-5-chat (LLM 调用)
   - running tool: get_weather (tool 执行)
   - GET geocoding-api.open-meteo.com (HTTP 请求)
   - GET api.open-meteo.com (HTTP 请求)

## 踩坑记录

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| Tempo `empty ring` 错误 | Tempo v2.10+ 默认启用 partition ring，单节点不兼容 | 使用 Tempo v2.7.2 |
| Loki `permission denied` | 容器内用户无权写 /var/loki | docker-compose 中设置 `user: "0"` |
| OTel Collector `loki` exporter 不存在 | 新版 collector-contrib 移除了 loki exporter | 改用 `otlphttp` exporter 发送到 Loki 的 `/otlp` 端点 |
| `OpenAIModel` 报错 | pydantic-ai 重命名为 `OpenAIChatModel`，参数改为 `provider` | 使用 `OpenAIChatModel` + `OpenAIProvider` |

## 方案对比

| 维度 | otel-tui | Logfire 平台 | Grafana 全家桶 |
|------|----------|-------------|---------------|
| 部署复杂度 | 低（1 容器） | 无（SaaS） | 中（4 容器） |
| 查看方式 | 终端 | Web | Web |
| Traces | 支持 | 支持 | 支持 |
| Logs | 支持 | 支持 | 支持 (Loki) |
| Metrics | 不支持 | 支持 | 支持 (Prometheus) |
| 告警 | 不支持 | 支持 | 支持 |
| 权限管理 | 不支持 | 支持 | 支持 |
| 适用场景 | 开发调试 | 小团队/快速上手 | 企业生产环境 |
